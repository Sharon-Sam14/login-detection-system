Risk Scoring

Understanding how SecureWatch calculates risk scores for login attempts.

Risk Assessment Algorithm

Low Risk (0-39%)

  • • Known IP addresses
  • • Regular login times
  • • Trusted devices

Medium Risk (40-69%)

  • • New locations
  • • Unusual login times
  • • VPN/Proxy usage

High Risk (70-100%)

  • • Multiple failed attempts
  • • Impossible travel
  • • Known threat sources

Risk Factors Explained

Our AI analyzes over 50 factors. Here are some key indicators of risk:

Multiple Failed Attempts

Repeated login failures within a short time, indicating potential brute force attacks.

Geolocation Anomaly

Login attempts from locations that are geographically impossible given recent activity.

Suspicious IP Range

IP addresses from known malicious networks or anonymization services.

New Device Detected

Login attempts from previously unseen devices or browsers for the user account.

Alerts & Actions

Details on different alert types and the recommended actions for each.

Alert Types

Critical Alert

High-risk login attempt blocked automatically. Immediate investigation recommended.

Warning Alert

Medium-risk activity detected. Manual review suggested.

Info Alert

Notable activity that doesn't require immediate action but should be monitored.

System Alert

Updates about system status, maintenance, or configuration changes.

Recommended Actions

High-Risk Attempts

System automatically blocks. Investigate source IP, check for compromised accounts, and consider IP range blocking.

Medium-Risk Attempts

Review attempt details. Contact the user to verify legitimacy and monitor for suspicious patterns.

Frequently Asked Questions

How accurate is the risk scoring?

Our AI-powered risk scoring system has a 95% accuracy rate, continuously improving through machine learning and threat intelligence updates.

What happens when a login is blocked?

Blocked attempts are logged, the user receives a security notification, and administrators are alerted. Legitimate users can request access through secure channels.

Can I customize the risk thresholds?

Yes, administrators can adjust risk thresholds, configure custom rules, and set up organization-specific security policies through the settings panel.

How is geographic data collected?

We use IP geolocation services combined with device fingerprinting to determine login locations while respecting user privacy and data protection regulations.

What integrations are available?

SecureWatch integrates with major identity providers, SIEM systems, and security tools through REST APIs and webhooks for seamless security operations.
